Отлично, задача ясна: три типа токенов: общий, индивидуальный и специальный для анонимных гостей. Анонимный токен позволяет войти без ввода имени/фамилии/должности, автоматически даёт роль «тайный гость».

Ниже — промпт для Replit. Отправьте его.

---

Промпт: Добавить тип токена «Анонимный гость»

1. Изменения в форме создания совещания

Добавить новый тип токена: «Анонимный гость» (отдельный чекбокс или радиокнопка).

· Если включено, система генерирует отдельный токен (например, ANON7X9Z) и сохраняет его в таблице meetings в поле anonymousToken.
· Этот токен не должен совпадать с общим токеном и не должен генерироваться в наборе индивидуальных токенов.

Интерфейс:

```
Тип токена:
◉ Общий (один для всех)
○ Индивидуальные (каждому свой)
☑ Анонимный гость (отдельный токен для входа без данных)
```

Если выбран «Общий» или «Индивидуальные», анонимный гость можно добавить как опцию.

2. База данных

В таблицу meetings добавить поле:

```sql
anonymousToken text
```

3. Страница входа

· При открытии /meet/:meetingId проверяется наличие query-параметра token.
· Если переданный токен совпадает с anonymousToken из БД:
  · Скрыть поля «Имя», «Фамилия», «Должность».
  · Показать только кнопку «Войти анонимно».
· При нажатии:
  · Создать запись в meeting_participants с isAnonymous = true, name = null, lastName = null, position = null, role = 'participant'.
  · Присвоить номер (например, максимальный + 1) и отображать в списке участников как «Гость {номер}» (или просто «Гость»).

4. Ограничения для анонимного гостя

· Не может отправлять файлы (кнопка скрепки отключена или скрыта).
· Не может рисовать на доске (даже если ведущий).
· Может поднимать руку (кнопка есть), при этом его кружок становится красным, ведущий видит «Гость поднял руку».
· Может писать текстовые сообщения в чат.

5. Отображение в списке участников

Вместо имени и фамилии — «Гость» (или «Гость 7», если нумерация). Иконка анонима (👤) и пунктирный ободок.

6. Безопасность и уникальность

· anonymousToken генерируется случайной строкой (6-8 символов, буквы+цифры).
· Проверять, что он не совпадает с commonToken и не входит в список индивидуальных токенов (если индивидуальные хранятся в отдельной таблице).

7. Пример работы

1. Создатель комнаты включает опцию «Анонимный гость». Система генерирует токен ANON3F9.
2. Создатель копирует ссылку .../meet/123456?token=ANON3F9 и отправляет гостю.
3. Гость открывает ссылку, видит только кнопку «Войти анонимно», нажимает её.
4. Гость попадает в комнату, все видят «Гость» в списке участников.

8. Обратная совместимость

Если в существующих комнатах нет поля anonymousToken, оно должно быть NULL. При попытке войти с токеном, которого нет в commonToken или в таблице индивидуальных токенов, и anonymousToken не совпадает — отказ.
